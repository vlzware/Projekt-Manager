# Secrets manifest — keys expected inside `secrets.env.age` (ADR-0012).
#
# One key per line. Lines starting with `#` are comments. Blank lines
# ignored. The manifest is the authoritative list the drift checks
# consume:
#
#   * `scripts/check-env-drift.sh` (CI): every compose-interpolated var
#     without a `:-default` (the "operator must supply" set) must appear
#     here OR in `.env.production.example`. Any key missing from both
#     fails CI.
#   * `scripts/deploy.sh` (pre-flight): every key in this manifest must
#     be present in the decrypted `secrets.env.age` before the deploy
#     proceeds. A missing key aborts the deploy with the exact list, no
#     containers are recreated.
#
# See `docs/ops/manual-deploy.md § Secrets` for per-key purpose and
# rotation procedure. Keep that prose and this list in sync — CI will
# also fail if a key is added here but not documented there (future).
#
# AGE_RECIPIENT is listed here because it lives in `secrets.env.age`
# operationally (co-managed with the private identity, rotated together)
# even though the recipient itself is a public age key, not cryptographic
# secret material.
