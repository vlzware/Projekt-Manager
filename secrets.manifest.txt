# Secrets manifest — keys expected inside `secrets.env.age` (ADR-0012).
#
# One key per line. Lines starting with `#` are comments. Blank lines
# ignored. The manifest is the authoritative list the drift checks
# consume:
#
#   * `scripts/check-env-drift.sh` (CI): every compose-interpolated var
#     without a `:-default` (the "operator must supply" set) must appear
#     here OR in `.env.production.example`. Any key missing from both
#     fails CI.
#   * `scripts/deploy.sh` (pre-flight): every key in this manifest must
#     be present in the decrypted `secrets.env.age` before the deploy
#     proceeds. A missing key aborts the deploy with the exact list, no
#     containers are recreated.
#
# See `docs/ops/manual-deploy.md § Secrets` for per-key purpose and
# rotation procedure. Keep that prose and this list in sync — CI will
# also fail if a key is added here but not documented there (future).
#
# AGE_RECIPIENT is listed here because it lives in `secrets.env.age`
# operationally (co-managed with the private identity, rotated together)
# even though the recipient itself is a public age key, not cryptographic
# secret material.

# Layer 1 (app runtime)
POSTGRES_PASSWORD
# Capability-restricted object-storage app-key secret (ADR-0022). The
# `applicationKey` half of the B2 key created via
#   b2 key create … readFiles,writeFiles,listFiles
# The matching `keyId` (STORAGE_ACCESS_KEY) is non-secret operator
# config in .env.production.example; this is the value compose forwards
# as STORAGE_SECRET_KEY into the app container. A missing value here
# trips deploy.sh's manifest pre-flight before any container starts.
# Rotation procedure: docs/ops/object-storage-provisioning.md (§ App key).
STORAGE_SECRET_KEY
CLOUDFLARE_API_TOKEN

# Web Push / VAPID. Private half of the P-256 ECDSA key pair — the
# server derives the public half at boot. Encrypt at rest because leaking
# the private key lets a third party send pushes that the browser will
# accept as if originating from this deployment. See
# docs/ops/troubleshooting.md (VAPID) and docs/ops/manual-deploy.md
# § Rotate a secret.
VAPID_PRIVATE_KEY

# OpenRouter LLM extraction (ADR-0016). Paid API token — encrypt at rest
# so a checkout-of-deploy-tree leak doesn't expose billed credentials.
# Optional: feature is disabled when the key is absent (no manifest
# entry would skip the deploy.sh pre-flight, so list it here and let
# the operator commit-out / leave empty if they don't use the feature).
OPENROUTER_API_KEY

# Layer 2 (off-site backup — ADR-0020)
R2_ACCESS_KEY_ID
R2_SECRET_ACCESS_KEY
R2_ENDPOINT
AGE_RECIPIENT
